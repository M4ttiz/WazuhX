# Repository Structure: WazuhX

- `backend/` - Node.js Express backend (API, business logic, integrations)
- `frontend/` - React frontend (UI, views, state)
- `prompts/` - Conversation prompt markdown, examples, configs
- `deploy/` - Docker, scripts, and production setup
- `assets/` - Logos, screenshots, images for branding and docs
- `scripts/` - Automation/utility scripts
- `examples/` - Example queries, API calls, or test cases
- `docs/` - Full documentation (architecture, API, deployment, etc.)
- `.env.example` - Sample configuration/settings for `.env`
- `.github/` - GitHub configs (templates, workflows)
- `.editorconfig` - Coding style rules for editors
- `docker-compose.yml` - Compose full stack services
- `README.md`, `CHANGELOG.md`, etc. - Root documentation entrypoints
